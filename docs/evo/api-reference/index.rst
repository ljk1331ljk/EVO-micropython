.. currentmodule:: evo

Evo API reference
=================

The ``EVOX1P`` and ``EVOX1E`` firmware builds are ESP32-S3 MicroPython builds
with two Evo-specific API layers:

* the native ``evo`` C module from ``ports/esp32/usermod``; and
* the frozen Python modules from ``ports/esp32/frozen/evo`` plus the
  board-specific frozen ``pins`` module.

The board manifests freeze the same Evo Python library set for both boards.
The board ``pins`` module is different for ``EVOX1P`` and ``EVOX1E`` and
defines the physical pin, I2C bus, PWM channel and motor-port mapping used by
the shared modules.

Native ``evo`` module
---------------------

Import with::

    import evo

System functions
~~~~~~~~~~~~~~~~

.. function:: evo_init(start_ble=True, console=True, max_chunk=160, debug=False)

   Initialise Evo runtime services. When ``start_ble`` is true the firmware
   imports the frozen BLE/download manager and starts it using the configured
   BLE name. ``console`` enables BLE console forwarding, ``max_chunk`` controls
   BLE transfer chunking where supported, and ``debug`` enables debug output.

.. function:: log(*args, sep=" ", end="\\n", stream="stdout")

   Print a message to the serial console and forward the same text to the BLE
   console if it is running.

.. function:: get_name()
              get_ble_name()
              get_wifi_name()
              get_controller_type()
              get_config()

   Read the saved Evo configuration. ``get_config()`` returns a dictionary
   containing ``name``, ``ble_name``, ``wifi_name`` and ``controller_type``.

.. function:: set_name(name, reset=True)
              set_ble_name(name, reset=True)
              set_wifi_name(name, reset=True)
              set_controller_type(controller_type, reset=False)

   Update saved configuration and return ``True`` on success. Names may contain
   letters, numbers, ``_`` and ``-``. The name setters reset the board by
   default after writing the configuration.

.. class:: EvoMotor(port=M1, type=GENERICWITHENCODER, flip=False)

   Native motor object for one DC motor port. ``port`` is one of ``M1`` to
   ``M4``. ``type`` selects the encoder preset and counts-per-rotation value.
   ``flip`` reverses the effective port wiring and encoder direction.

   .. method:: run(power)
               runPower(power)

      Drive the motor with raw PWM power. Positive and negative values select
      direction. ``run()`` is a compatibility alias for ``runPower()``.

   .. method:: runSpeed(target_dps)

      Run closed-loop speed control towards ``target_dps`` degrees per second.

   .. method:: runTime(dps, seconds)

      Run closed-loop speed control for ``seconds`` seconds, then stop.

   .. method:: runAngle(dps, angle)

      Run until the encoder has moved ``angle`` degrees. The sign of ``angle``
      selects direction; ``dps`` is treated as a magnitude.

   .. method:: brake()
               coast()
               hold()
               setStopBehaviour(behaviour)
               setStopBehavior(behaviour)

      Stop or configure stopping behaviour. ``behaviour`` is ``COAST``,
      ``BRAKE`` or ``HOLD``.

   .. method:: getPosition()
               resetPosition()
               getAngle()
               resetAngle()

      Read or reset the encoder position. ``getPosition()`` returns encoder
      counts; ``getAngle()`` returns degrees derived from counts per rotation.

   .. method:: setCountsPerRotation(counts)
               flipEncoderDirection()

      Adjust encoder conversion and direction.

   .. method:: updateSpeed()
               getSpeed()
               getSpeedCPS()
               getSpeedDPS()
               getPower()
               setSpeedPID(kp, ki, kd)
               setSpeedLimits(min_power, max_power)
               resetSpeedControl()

      Closed-loop speed helpers. ``getSpeed()`` is degrees per second,
      ``getSpeedCPS()`` is counts per second, and ``getPower()`` returns the
      latest PWM power generated by speed control.

   .. method:: enableAllMotors()
               disableAllMotors()
               enableAll()
               disableAll()

      Control the shared motor-driver sleep line when the board ``pins`` module
      defines ``NSLEEP_PIN``.

   .. method:: deinit()

      Stop according to the configured stop behaviour and detach the encoder
      interrupt handler.

.. class:: EvoMotorPair(m1, m2)

   Native synchronized two-motor drive object. ``m1`` and ``m2`` are
   :class:`EvoMotor` instances.

   .. method:: setStartSpeed(speed)
               setEndSpeed(speed)
               setAcceleration(acceleration)
               getAcceleration()
               setDeceleration(deceleration)
               getDeceleration()

      Configure speed ramping for timed and encoder-distance moves.

   .. method:: setAccelerationProfile(profile)
               getAccelerationProfile()

      Select ``ACCEL_NONE``, ``ACCEL_TRAPEZOID`` or ``ACCEL_SCURVE``.

   .. method:: setStopBehaviour(behaviour)
               setStopBehavior(behaviour)
               setSyncPID(kp, ki, kd)
               getSyncPID()

      Configure default stop behaviour and encoder synchronization PID.

   .. method:: move(left_dps, right_dps)
               moveSpeed(left_dps, right_dps)
               movePower(left_power, right_power)

      Start continuous movement. ``move()`` is a speed-control alias for
      ``moveSpeed()``; ``movePower()`` uses raw PWM power.

   .. method:: moveDegrees(left_dps, right_dps, degrees, stopBehavior=None)
               runDegrees(left_dps, right_dps, degrees, stopBehavior=None)

      Run both motors for an encoder distance and then stop.

   .. method:: moveTime(left_dps, right_dps, time_ms, slowdown_ms=200, stopBehavior=None)

      Run both motors for ``time_ms`` milliseconds, optionally slowing down
      during the final ``slowdown_ms`` milliseconds.

   .. method:: hold()
               brake()
               coast()
               stop(stopBehavior=BRAKE)
               resetAngle()
               isBusy()
               deinit()

      Stop, reset or query the pair.

.. class:: EvoMecanum(frontLeft, frontRight, rearLeft, rearRight)

   Native synchronized four-motor mecanum drive object. All arguments are
   :class:`EvoMotor` instances.

   .. method:: setStartSpeed(speed)
               setEndSpeed(speed)
               setAcceleration(acceleration)
               getAcceleration()
               setDeceleration(deceleration)
               getDeceleration()
               setAccelerationProfile(profile)
               getAccelerationProfile()
               setStopBehavior(behaviour)
               setStopBehaviour(behaviour)
               setSyncPID(kp, ki, kd)
               getSyncPID()

      Configure ramping, stop behaviour and wheel synchronization.

   .. method:: move(x_dps, y_dps, turn_dps)
               moveSpeed(x_dps, y_dps, turn_dps)
               movePower(x_power, y_power, turn_power)

      Start continuous mecanum motion. ``x`` is strafe, ``y`` is forward/back,
      and ``turn`` is rotation. ``move()`` is a speed-control alias for
      ``moveSpeed()``.

   .. method:: moveDegrees(x_dps, y_dps, turn_dps, degrees, stopBehavior=None)
               runDegrees(x_dps, y_dps, turn_dps, degrees, stopBehavior=None)

      Run a mecanum move until the encoder distance target is reached.

   .. method:: moveTime(x_dps, y_dps, turn_dps, time_ms, slowdown_ms=200, stopBehavior=None)
               runTime(x_dps, y_dps, turn_dps, time_ms, slowdown_ms=200, stopBehavior=None)

      Run a mecanum move for a fixed time.

   .. method:: hold()
               brake()
               coast()
               stop(stopBehavior=BRAKE)
               resetAngle()
               isBusy()
               deinit()

      Stop, reset or query the mecanum drive.
